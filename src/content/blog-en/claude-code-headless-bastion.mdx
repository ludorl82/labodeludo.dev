>
> -   **Context**: Claude Code runs day to day inside the console container on my bastion, in a persistent tmux session.
> -   **Goal**: get it to run recurring tasks (backups, system upgrades) without me having to open an interactive session.
> -   **Solution**: scheduled tasks start a Claude Code instance in headless mode on the bastion; the remote control feature lets me check in on or steer a running session from my phone when needed, no SSH terminal required.
> -   **Cost**: moved from pay-per-use billing (up to $100 in a single heavy day) to the Claude Pro subscription, $40/month, covering both interactive sessions and headless work.

For several months now, Claude Code has been my main tool for everything infrastructure-related in my home lab. What's changed recently is that I'm no longer the only one triggering it: part of that work now runs entirely on its own, without me sitting down at a keyboard.

## The bastion: a container, not the host machine

I've already described [my containerized console](/en/blog/compartimentalisation-des-outils-de-console/) elsewhere: rather than installing my development tools directly on my bastion (the SSH entry point of my server network), I run them in a container defined as code, rebuildable identically anywhere. Claude Code runs inside that container — not on the host itself.

## A session that lives in tmux

Inside that container, I keep Claude Code in a `tmux` session that runs continuously. The advantage is that the session — its context, its history, the state of whatever it's doing — survives a disconnect. I can close my terminal, come back an hour later, reattach to the `tmux` session, and pick up exactly where I left off, as if I'd never left.

## Remote control, no SSH terminal required

Claude Code has a remote control feature that lets me check in on, and if needed steer, a running session from my phone — no need to open an SSH client or manually reattach to the bastion's `tmux`. It's especially useful for a headless task running unsupervised: I can glance at what it's done, or give it a new instruction, without sitting down at a computer.

## Headless mode: tasks that run without me

This is the real shift in scale. A Claude Code instance started with no interface at all — "headless" mode, kicked off from a scheduled task on the bastion — now handles two things every week, at night, while everything sleeps:

-   **Encrypted backups** of every machine in my lab, pushed to off-site storage.
-   **System upgrades** across my whole fleet: it connects to each machine, checks what needs updating, applies the patches, and documents what it did.

It's the same agent, the same reasoning and command-execution loop as my interactive sessions — just triggered on a schedule instead of by me. I don't have to think about it anymore.

## The economics: a subscription instead of tokens

A word on cost, because the question comes up often when I talk about this. For a long time I ran Claude Code on pay-per-use API billing — convenient to get started, but it can add up fast: a single heavy work day once cost me around $100 in tokens alone. I've since switched to the Claude Pro subscription, at $40 a month, which covers both my interactive sessions and the headless work described above. For usage as sustained as mine, the flat monthly subscription comes out well ahead of metered billing — and it completely removes the anxiety of checking the bill after a big work session.

## What we're taking away

-   A containerized bastion gives a portable, rebuildable environment instead of a pile of hand-configured state on the host.
-   Keeping a session in `tmux` lets me reattach any time without losing context.
-   Remote control removes the need for an SSH terminal just to check on or steer a running task.
-   The same agent that helps me in interactive sessions can run unsupervised, on a schedule, for recurring tasks like backups or system upgrades.
-   If usage is sustained, a flat subscription (Claude Pro, $40/month) beats pay-per-token billing by a wide margin — I've burned $100 in a single day on API alone.
